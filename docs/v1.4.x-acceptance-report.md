## 4. Outline `level='book'` invariant + auto-save UPSERT (A4.1)

**Goal**: the DB layer must enforce “one book outline per project” so the A5 query path (`SELECT … WHERE level='book'`) cannot raise `MultipleResultsFound` and cannot read stale rows.

**Bugs found while observing D‑2 deeply**:

- **Bug F** — `knowledge_tasks.py:383` `level = task.task_type.replace("outline_","")` produced `level='from_reference'`, which the A5 query (hard-coded `level='book'`) could never see.
- **Bug G** — the `outlines` table had no `(project_id, level)` uniqueness; stale rows accumulated and `scalar_one_or_none()` raised `MultipleResultsFound` on duplicates.
- **Bug H (worker event-loop noise)** — surfaced by full-log grep (17× `Event loop is closed` + 13× `Future attached to a different loop` + 2× `never awaited` over the day, all from `retry_reference_book_missing_branches`). Tracked but **not** in scope for A4.1; see point 6.
- **Bug I** — the `outline_volume` auto-save branch in the same writer was missing `parent_id` and would create orphan rows; declared dead-code (the real volume save happens in `volumes.py`).
- **Bug J (related)** — D‑2 wrote `is_confirmed=0` by default; resolved at the A5 step via the `POST /api/outlines/{id}/confirm` endpoint.

**Fix** (commit `a03b4f7 fix(outlines): enforce one-book-per-project invariant + upsert auto-save`):

- alembic `a1001500_v150_outline_book_unique`:
  1. `UPDATE outlines SET level='book' WHERE level='from_reference'` (semantic correction).
  2. `ROW_NUMBER() PARTITION BY project_id ORDER BY created_at DESC` keeps newest book row per project; deletes the rest (26 → 17 rows, 9 stale removed).
  3. `CREATE UNIQUE INDEX outlines_project_book_unique ON outlines(project_id) WHERE level='book'` (partial UNIQUE).
- `knowledge_tasks.py:380-388` rewrites the auto-save with an explicit `_outline_level_map = {"outline_from_reference":"book","outline_book":"book"}` and a write-time `delete(Outline).where(project_id, level='book')` UPSERT; `outline_volume` tagged dead-code.

**A6 re-check (2026-04-26 13:50Z)**: outlines table = `9 book + 9 volume = 18 rows`; `SELECT project_id, count(*) FROM outlines WHERE level='book' GROUP BY project_id HAVING count(*)>1` returns 0 rows. Partial UNIQUE index in place. 0 `UniqueViolation / duplicate-key` since `a03b4f7`.

**Status**: ✅ PASS.

## 5. Volume regenerate + chapter SSE auto-save (A5 + Bug K)

**Goal**: based on the confirmed book outline, the volume regenerate SSE produces volume + chapter outlines, the chapter generate SSE streams text **and persists it to PG**.

**A5 walk-through** (project `f14712d6-6dc6-4cfb-b05f-e107fa02b63d` “验收测试-玄幻全本200万”):

1. Confirm book outline `453fca6e-016b-49b5-82f5-b6d4cb978666` via `POST /api/outlines/{id}/confirm` → `is_confirmed=1`.
2. Create volume1 `eeeb4a68-3eab-4057-a102-fbe460ca2bb7` (`HTTP 201`).
3. Volume1 regenerate SSE — 110s, `73982 bytes`, 2 outlines + 15 chapters created (status=draft), 0 noise. Evidence `.tmp_a5_vol1_regen.sse`.
4. Chapter1 SSE (`932922e8-2055-46ce-8673-8ce8111e0108`) — first run `183701 bytes / 6826 chunks / status:completed + [DONE]`, **but `chapters.content_text` stayed 0 bytes / status=draft**. Evidence `.tmp_evidence_a5_ch1_completed_DBmiss.sse`. **Bug K exposed.**

**Bug K root cause**: `backend/app/api/generate.py:54 generate_chapter` SSE handler had no auto-save path — `async for chunk: yield ...; status:completed; [DONE]` only — while `generate_outline:184-355` in the same file already had a fully-formed `collected_text + async_session_factory + UPSERT + status:saved` pattern. Same omission family as Bug F / Bug I.

**Fix** (commit `662ceeb fix(generate): persist chapter content on SSE completion (Bug K)`):

- `event_stream()` top: `collected_text: list[str] = []`.
- Inside the chunk loop: `if chunk: collected_text.append(chunk)` before `yield`.
- Before `status:completed`: open an independent `async_session_factory()`, prefer `await save_db.get(Chapter, req.chapter_id)`, fall back to `select(Chapter).where(volume_id, chapter_idx).scalar_one_or_none()`, UPDATE `content_text / word_count / status="completed"`, commit, then yield `{"status":"saved","chapter_id":…,"word_count":…}`. `except` branch is `logger.warning` so SSE never aborts on save error.
