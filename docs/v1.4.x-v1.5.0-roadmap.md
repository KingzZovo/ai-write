## Execution rules

1. **Sequential**: A1 → A2 → A3 → A4 → A5 → A6 → B1 → B2 → C1 → C2 → C3 → C4. No parallel work unless King explicitly authorizes a fork.
2. **Each step must end with**: `[ID] DONE` log line in this file's status column + git commit (when code changed) + verification output pasted in the chat.
3. **Discoveries that block forward progress**: open as new sub-tasks under the current phase, not silently merged into the next.
4. **Acceptance gates are hard**: do not mark `done` if verification gate is not green. Mark `partial` and explain.
5. **King's mandates already in effect**:
   - 主动诊断（看 logs、对账业务真值表，不只 status=success）
   - succeeded ≠ healthy
   - row count ≠ content validity
   - <2-min sample 不下"已恢复"结论
   - dependency change → must rebuild image
   - 0 LLM call + 0 HTTP POST → 必看 RuntimeError 栈
   - `_slices_missing_*` query 必须检查内容质量，不只看行存在
