## P0 Bug Discovered & Fixed

### 现象

修前调 `POST /evaluate`：
```
HTTP 200 in 0.008s
body: {"...": "Evaluation error: No model configured for 'evaluation'. Configure at Settings > Model Configuration."}
```

llm_call_logs 中**没有任何 evaluation 行**。所有 evaluation 行 score=0.0。

### 根因

`ChapterEvaluator.__init__` 在同步上下文中调用 `get_model_router()`，拿到的 router 实例 `_db_loaded=False`、`providers={}`。`evaluate()` 后续再调 `router.generate(task_type="evaluation", ...)` 直接报 `No model configured`。

consistency_checker / ooc_checker 同样问题。

**影响**：
- evaluator 自上线起从未做过实际评分
- 三个 checker 的 LLM 兜底也从未生效
- llm_call_logs 不存在该来源的行 → 监控/计费均无感

### 修复（B1' Step B）

三个文件迁移到异步 lazy router：

| 文件 | 改动 |
|------|------|
| `backend/app/services/chapter_evaluator.py` | `__init__` 不再 prefetch router；`evaluate()` 改 `await get_model_router_async()` + `generate_with_tier_fallback(task_type="evaluation", temperature=0.3, max_tokens=4096, _log_meta={...})` |
| `backend/app/services/checkers/consistency_checker.py` | `_llm_consistency_check` 改 async + tier_fallback (`task_type="extraction"`, max_tokens=1024) |
| `backend/app/services/checkers/ooc_checker.py` | `_llm_ooc_check` 改 async + tier_fallback (`task_type="extraction"`, max_tokens=1024) |

并在 `ModelRouter` 增强 fallback 链：
- 新增 `_pick_endpoints_by_tier(tier) -> list[str]` 返回 tier 内**所有** chat-capable endpoint（不再只取第一个）
- `_build_tier_attempts` 末尾加 chat-capable 兜底链：把任意未在已游走 tier 的非 embedding endpoint 也加入尝试队列
- 双重去重保证不重复尝试同一 endpoint

## Transient Upstream Error (mitigated)

第二次 baseline 跑（before fallback enhancement）：
```
APIError: stream error: stream ID 1; INTERNAL_ERROR; received from peer
```
来源：本地 Qwen HTTP/2 streaming 偶发 reset。  
第三次跑（after fallback enhancement）：同一 endpoint 一次成功。  
→ Fallback enhancement **降低单点瞬态故障对用户的暴露**：下次再发生 stream reset 时，`_build_tier_attempts` 会自动切到 `ac6eb9cd...` (大纲 endpoint, 同 standard tier) 重试。
